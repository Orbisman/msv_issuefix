The Sun XML Schema Part 2 Java Implemantation (XSDLib) is an implementation of W3C XML Schema Part 2, intended for use by application developers as their building block.

This preview version implements 31st,Jan,2001 version of the candidate recommendation, which is only available to W3C members. As its date suggests, this version is newer than publicly available 24th,Oct,2000 edition.

W3C XML Schema Part 2 is still a candidate recommendation, which means it is subject to change. In fact, many important changes are made from 24th,Oct,2000 edition and many changes are still likely to be made before it becomes a recommendation. So developers should be warned for possible future changes including (but of course not limited to)

  • change/addition/removal of type names
  • change/addition/removal of facets

Known Limitations

  1. For "float" and "double" types. The spec says a lexical value must be mapped to the closest value in the value space, but XSDLib cannot accept values which are bigger than maximum value of that type or smaller than minimum value of that type.
  2. "duration" facet and "period" facet are not implemented because they are removed.
  3. "binary" type and "encoding" facet are not implemented because they are removed. Instead, "base64Binary" type and "hexBinary" types are implemented.
  4. "ID", "IDREF", and "IDREFS" types are not implemented. These types are so special that it is impossible to provide a generic implementation. com.sun.tranquilo.datatype.DataTypeFactory does not recognize these three types.
  5. "CDATA" type is implemented as "normalizedString" type, again because it is renamed as such.
  6. "NOTATION" type is not implemented. com.sun.tranquilo.datatype.DataTypeFactory does not recognize this type name.
  7. "timeInstant" type is implemented as "dateTime" type.
  8. "timeDuration" type is implemented as "duration" type.
  9. "timePeriod" type is not implemented because it is removed.
  10. "century" type is not implemented because it is removed.
  11. "month" type is implemented as "yearMonth" type.
  12. "recurringDate" type is implemented as "monthDay" type.
  13. "recurringDay" type is implemented as "day" type.
  14. "length", "minLength", and "maxLength" facet is effectively limited upto 2147483647. Values above this limit are recognized, but will be treated as this limit. Items longer than this limit will not be validated correctly.
  15. Implementation of "precision" and "scale" should be considered as incomplete. The spec is not clear enough.
