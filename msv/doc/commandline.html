Sun Tranquilo Verifier is a Java tool to validate XML documents with a schema. It supports RELAX Namespace, RELAX Core, and TREX.

Usage

To validate XML documents with a RELAX grammar/module, do as follows 

$ java -jar tranquilo.jar MySchema.rxg MyDocument1.xml MyDocument2.xml ...

If you'd like to use TREX, 

$ java -jar tranquilo.jar MySchema.trex MyDocument1.xml MyDocument2.xml

Or if you'd like to use XML Schema, 

$ java -jar tranquilo.jar MySchema.xsd MyDocument1.xml MyDocument2.xml

Tranquilo will detect the schema language regardless of its file extension.

However, you have to add -dtd switch to use DTD. 

$ java -jar tranquilo.jar -dtd my.dtd MyDocument1.xml MyDocument2.xml

Some environment supports wild card for filenames, while others don't (e.g., "jview" VM from Microsoft). You can also use URL instead of filename.

If the schema contains some errors, you will see messages like this. 

start parsing a grammar.
the following hedgeRules form a infinite recursion ( y > z > y )
  18:23@file:///c:/work/relax/relax024.e.rlx
  14:23@file:///c:/work/relax/relax024.e.rlx
failed to load a grammar.

"18:23@XYZ" indicates that the error is related with line 18, column 23 of file XYZ.

If the document is valid, you will see messages like this.

start parsing a grammar.
validating c:\work\relax\relax001.v00.xml
the document is valid.

Alternatively, if the document has some errors, you will see messages like this.

start parsing a grammar.
validating c:\work\relax\relax001.n02.xml
Error at line:5, column:5 of file:///c:/work/relax/relax001.n02.xml
  tag name "q" is not allowed. possible tag names are:

the document is NOT valid.

Please note that line/column infomation is sometimes inaccurate, and one tab char is counted as one character, not 4 or 8. So you might want to look around to find the actual error.

Options

Command line tool has several other options. You can see the list of available options by invoking Tranquilo without any arguments.

$ java -jar tranquilo.jar
-warning
Display all warning messages. Warnings are useful to catch potential errors.
-loose
This switch prevents Tranquilo to resolve external resources (such as external DTD referenced from documents, or external entities).

Known Limitations

DTD

  1. Attribute declarations in the form of xmlns:*** are ignored. All namespce declarations within the instance document are also ignored.
  2. In general, handling of XML namespace is different from normal DTD validators. As a result, Tranquilo may validate documents that other DTD validators reject. If there is such a danger, Tranquilo always reports a warning.

TREX

This implementation fully conforms to the current specification.

RELAX Core

This implementation fully conforms to the current specification.

RELAX Namespace

  1. <anyOtherAttribute> is not implemented.

XML Schema Part 1

  1. "Schema component constraints" and "schema representation constraints" are not fully enforced. In other words, Tranquilo accepts schemata that are rejected by other conforming processors.
  2. "Identity constraints" are not implemented. key, keyref and unique declarations are ignored and no check is performed.
  3. Forward reference of simple type is not supported. Therefore, the following schema cannot be used. 
    
<xs:simpleType name="foo">
 <xs:restriction base="bar">
  <xs:fractionDigits value="2"/>
 </xs:restriction>
</xs:simpleType>

<xs:simpleType name="bar">
 <xs:restriction base="xs:decimal">
  <xs:totalDigits value="5" />
 </xs:restriction>
</xs:simpleType>
  4. "Missing sub-components" are treated as immediate error. The present author believes this behavior better serves the majority.
  5. Default values are ignored. In fact, no infoset contribution is supported: Tranquilo validates documents, but it doesn't touch documents. PSVI is also not supported.
